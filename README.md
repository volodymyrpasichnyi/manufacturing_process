Виробничі процеси для будь‑якого продукту схожі на складання пазлу: вироби створюються крок за кроком, і дуже важливо уважно стежити за кожним етапом. У цьому проєкті команда прагне покращити спосіб моніторингу та контролю виробничого процесу. Мета полягає у впровадженні більш методичного підходу, відомого як статистичний процес‑контроль (SPC). SPC — це усталена стратегія, яка використовує дані для визначення того, чи працює процес належним чином. Коригування процесу здійснюється лише тоді, коли вимірювання виходять за межі допустимого діапазону.
Цей допустимий діапазон визначається верхньою контрольною межею (UCL) та нижньою контрольною межею (LCL), формули яких для вибірки з п’яти виробів мають вигляд:

UCL = \bar{x} + 3 * \frac{\sigma}{\sqrt{5}}
LCL = \bar{x} - 3 * \frac{\sigma}{\sqrt{5}}

UCL визначає найбільшу допустиму висоту деталей, тоді як LCL визначає найменшу допустиму висоту. Ідеально, якщо всі вироби потрапляють у межі між цими двома значеннями. Використовуючи SQL‑віконні функції та вкладені запити, необхідно проаналізувати історичні виробничі дані, щоб визначити цей допустимий діапазон і знайти будь‑які точки процесу, які виходять за межі та потребують коригування. Це забезпечить стабільний виробничий процес, який постійно виготовляє продукцію високої якості.

Дані для аналізу доступні в таблиці manufacturing_parts, яка містить такі поля:
- item_no — номер виробу
- length — довжина виробу
- width — ширина виробу
- height — висота виробу
- operator — оператор (виробнича машина)
